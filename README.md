# Memee

## 概要
Memeeはweb上で動作するTodoリストアプリケーションです。  
動作確認用ページ(Render.com): [https://memee-nunp.onrender.com](https://memee-nunp.onrender.com)  
Render.comの無料枠を使用しているため、起動に数分かかる可能性があります。

## 使用技術
- 使用言語: PHP 8.4.1, JavaScript
- バックエンドフレームワーク: Laravel 12.x
- ビルドツール: vite
- フロントエンド: Blade, tailwindcss v4, Alpine.js, Axios
- データベース: PostgreSQL 18

## 機能

- タスクの登録  
締切日と説明を決めた後に登録ボタンを押すとタスクを登録できます。登録したタスクは蓄積され、確認することができます。  
説明は最大100文字までであり、空にすることはできません。

- タスクの表示  
タスクは画面左側に一覧として表示されます。説明と締め切り、完了状況以外に残り日数も表示されます。残り日数は締切日の午前0:00までの時間を表しています。締め切りを超過した場合には経過日数が表示されます。  
また、タスクは締め切りが近い順で表示されます。

- タスクの管理  
タスクは「完了！」と「未完了」の二種類の状態をとります。 タスクの右側にあるボタンで変更できます。

- タスクの削除  
完了、未完了に関わらずタスクは削除することができます。タスクの右側にあるボタンで操作できます。  
タスクを削除すると仮削除が行われます。この状態になると「削除されたタスク」として分類され、通常の一覧画面には表示されなくなります。なお、時間経過によって自動的に削除されることはありません。  
仮削除されたタスクは完全に削除することが可能です。完全削除を行うとデータベースから完全にデータが消えます。

- 表示するタスクの切り替え  
「未完了のタスク」「完了したタスク」「削除されたタスク」の三種類でタスクは分けられます。これらの表示を切り替えることができます。

## 画面構成
基本的に画面はメイン画面の一つだけです。左側にはTodoリストが表示され、右側には表示切り替えボタンとタスク登録ボタンがあります。  
スマートフォンで表示した際にはタスク登録画面とタスク一覧画面の二つとなります。タスク登録画面を表示するには右下の追加ボタンを押し、元の画面に戻るには右上の閉じるボタンを押します。また、表示切り替えボタンはタスク一覧画面の上に配置されます。

## 工夫した点
- SPAライクな作りにより快適な動作を実現  
Todoリストの性質上、画面遷移が少ないためJavaScriptを使用して非同期に表示の制御およびデータの送受信を行う仕組みをとりました。  
サーバーサイドでレンダリングを行うため、完全なSPAではありませんが、画面遷移の際の通信やレンダリングは必要最低限なものだけとなっています。  
また、Bladeファイルの中に記述しても比較的読みやすくなるため、純粋なJavaScriptではなくAlpine.jsを使用しました。  

- レスポンシブ対応デザイン  
スマートフォンからでも使用できるようにデザインしました。  
※スマートフォンからアクセスした場合、タスク登録ボタンがタスクに被ることがありますが、余白や表示タスク切り替えの部分で上にスクロールすると見えるようになります。

- 手数の少ない操作感  
全体的に少ない手数で操作が行えるようにデザインしました。  
特にタスクの登録の際に必ず日付を選択するため、初めからカレンダーを展開して、手間を減らしました。また、カレンダーには使いやすさを重視してflatpickrというライブラリを使用しました。
